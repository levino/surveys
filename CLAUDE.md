# surveys — developer notes

Self-hosted form/survey collector. One Go binary + embedded SQLite + embedded
CSS. See `README.md` for the user-facing overview and configuration.

## Architecture

- **Public forms** (`public.go`, `ui/ui.templ`): anonymous HTML forms at
  `/f/{slug}`, `noindex`, honeypot + time-token + per-IP rate-limit. Submissions
  validated client-side (native HTML5 + `formValidation()` enhancer) and
  server-side (`validateSubmission` in `forms.go`, the source of truth).
- **MCP endpoint** (`mcp.go`, `tools.go`): the only authenticated surface. Tool
  definitions live in `toolDefs()`, dispatch in `callTool`.
- **Auth** (`oauth.go`, `oauth_routes.go`, `oidc.go`, `auth.go`): own OAuth 2.0
  AS for MCP clients (dynamic registration + PKCE); login delegated to an
  upstream OIDC provider. A survey is owned by a `groups`-claim team.
- **Persistence** (`db.go`): SQLite (modernc, CGO-free), single writer.
- **Markdown** (`md.go`): goldmark renders description + field `help`; raw HTML
  is escaped (not unsafe).

## Build / test

```bash
go test ./...                 # in-process OIDC mock; no network
go run .                      # needs assets/app.css (npm run build:css once)
templ generate                # regenerate ui/*_templ.go after editing *.templ
npm ci && npm run build:css   # rebuild Tailwind/DaisyUI CSS
```

`ui/ui_templ.go` is generated by [templ](https://templ.guide) and **committed**
(the Dockerfile does not run `templ generate`). Always re-run `templ generate`
and `gofmt` after touching `ui/ui.templ`.

The CSS (`assets/app.css`) is **not committed** — it is generated from
`assets/input.css` + scanning `ui/**/*.templ`, and embedded via `//go:embed`.

## Conventions

- Config is env-only (`config.go`); no config files.
- Keep the two surfaces minimal: no REST API, no admin UI beyond the dashboard.
